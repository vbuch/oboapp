# Fork and Upstream PR Workflow

Use this workflow when the change should land in the upstream repository rather than only in a fork.

1. Start the branch from `oboapp/oboapp:main`.
2. Make the changes in your forked workspace.
3. Push the feature branch to your fork.
4. Open the pull request against `oboapp/oboapp`.

Keep the branch name and PR title aligned with the upstream change so the same branch can be reviewed in the fork and merged upstream without extra rename steps.
